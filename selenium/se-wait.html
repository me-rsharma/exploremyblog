Let's explore Selenium wait using Python code

By Rajeshwar Sharma

As we all know, change is the only constant and the same goes with our very close companion ie. Internet connection. Sometimes, it happens we have fluctuating internet speed especially when people working from home 😀

In such a situation, the loading of web elements on webpages slows down and selenium sometime faces issues while interacting with them and return different sorts of exceptions such as element not visible element is not enabled element is not present and so on.

So to handle this scenario, we generally make selenium to wait until the web elements get loaded on the webpages completely. There are different ways we can make out the selenium code to wait.

Using Python sleep() function

Using sleep() function from time module of Python helps selenium to wait for specific interval of time

sleep() is a parameterized function that expect wait time arguments as int

The int argument passed to the sleep(5) function will be considered as SECONDS and our code will pause for that specific time

Usage of sleep() function in our python code

👉NOTE: Always make sure to import sleep function from time module before using as from time import sleep

Though sleep() does pretty well job, it's not recommended by the selenium community to use sleep() in our automation script. 🤔

Don't worry, it's not because of hatred or jealously of selenium towards python 😀 but due to strict behavior of sleep(). And somewhere it is right because people don't like strictness in their life 😁

A reason not to use sleep()

The strict behavior of Python sleep() is not much appreciated by selenium web driver because when we specify sleep(5) to make selenium wait till the element get loaded completely on the webpages, the function will forcibly pause the code execution for 5 seconds and do not allow selenium to move ahead with execution even though web elements get loaded before 5 seconds.

And I think this behavior of sleep() makes selenium angry and selenium develops its own function and classes for the wait

Yes, you heard me right we have selenium waits that are more efficient and reliable to use than sleep().

So let's dive into the selenium waits and see its importance and usage.

Types of Selenium Wait

Selenium wait majorly classified into 2 types:

Implicit Wait

Explicit Wait

Implicit wait & its usage

In Selenium, implicit wait helps to wait for a certain period of time that is given to the implicit wait method before throwing an exception. If we don't specify any time duration, its default setting is 0. Once we set implicit wait in our code, the implicit wait will be applicable throughout the lifetime of the drive or in simple terms for the entire duration for which the browser is open

To set implicit wait in Selenium using Python, we need to use implicitly_wait(5)

  • Usage of selenium implicit wait

Explicit wait & its usage

In Selenium, explicit wait behaves bit differently than implicit wait. Explicit wait helps to wait for a certain period of time that is given to the explicit wait method based on certain condition before throwing an exception. Hence, we must always use expected_conditions module along with WebDriverWait class

Also, in explicit wait we can set the polling_frequency and ignored_exceptions which defines the frequency with which WebDriver will check if the condition appears before throwing the exception and helps to ignore set of exceptions when element not get accessed within defined time respectively.

It's a good idea to set explicit wait where we have cases, elements take more time to load. If one sets an implicit wait command, then the browser will wait for the same time frame before loading every web element. This causes an unnecessary delay in executing the test script.

Once we set explicit wait in our code, the explicit wait will be applicable till the time condition is met or the time duration for which we have applied explicit wait

If we don't specify any time duration, its default setting is 0. the default polling_frequency is 0.5 and the default ignored_exceptions is None

To set explicit wait in Selenium using Python, we need to use WebDriverWait class
